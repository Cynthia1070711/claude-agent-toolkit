# Step 5: Web Research

**Goal:** Ensure latest technical knowledge to prevent outdated implementations.

---

## AVAILABLE STATE

- `{architecture_content}` — from Step 2 (for library/framework identification)

---

## EXECUTION SEQUENCE

> **CRITICAL:** 🌐 ENSURE LATEST TECH KNOWLEDGE — Prevent outdated implementations!

### 1. Identify Research Needs

From architecture analysis, identify specific libraries, APIs, or frameworks that require latest version knowledge.

**WEB INTELLIGENCE:**
Identify specific technical areas that require latest version knowledge.

### 2. Research Critical Technologies

For each critical technology, research latest stable version and key changes:
- Latest API documentation and breaking changes
- Security vulnerabilities or updates
- Performance improvements or deprecations
- Best practices for current version

### 3. Include External Context in Story

**EXTERNAL CONTEXT INCLUSION:**

Include in story any critical latest information the developer needs:
- Specific library versions and why chosen
- API endpoints with parameters and authentication
- Recent security patches or considerations
- Performance optimization techniques
- Migration considerations if upgrading

---

## SUCCESS METRICS

- Critical technologies researched for latest versions
- Breaking changes and security updates identified
- Relevant information ready for story Dev Notes injection

## FAILURE MODES

- Skipping research for fast-moving technologies
- Using outdated API documentation
- Not flagging security patches or deprecations
